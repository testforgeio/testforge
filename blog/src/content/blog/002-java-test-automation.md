Test automation frameworks simplify the process of ensuring your applications work as expected. It helps to colaborate with yout team and increase confidence of your releases.
In this guide, we’ll build a robust dedicated test automation framework using Java, Maven, Selenide, and Allure. 
By the end, you’ll also learn how to package your tests into a JAR file, run them in Docker, and configure a remote browser.

## Table of contents

## Create Maven Project

The first step is to create a [Maven](https://maven.apache.org/) project. 
Maven streamlines the build process by standardizing the project structure and managing dependencies via the Project Object Model (POM).

Key points in Maven project creation.

- **Using Maven Archetypes**:
  Maven archetypes provide ready-made templates to quickly scaffold a project. For example, the maven-archetype-quickstart archetype sets up a basic project with a standard directory layout and a sample pom.xml.

- **Project Structure**:
  Once generated, the project includes:
  - src/main/java for your test steps and helper classes.
  - src/test/java for your testing code.
  This structure helps keep common utility code separate from test code and keeps the project organized.

- **Customizing the POM**:
  The pom.xml file, generated by Maven, defines your project’s groupId, artifactId, version, dependencies, and plugins.
  We will modify this file to add libraries, plugins, and configure the build process.

- **IDE Integration**:
  Modern IDEs like IntelliJ IDEA and Eclipse offer seamless Maven integration. They allow you to import a Maven project directly, automatically recognizing the project structure and dependencies.

## Steps to create a Maven project:

Change the groupId and artifactId to your project name. Usually the groupId is reversed domain name of your company.
ArtifactId is the name of your project.

```bash
mvn archetype:generate -DgroupId=blog.testforge -DartifactId=testframework -DarchetypeArtifactId=maven-archetype-quickstart -DinteractiveMode=false
```

Example pom.xml snippet:

```xml
<project xmlns="http://maven.apache.org/POM/4.0.0" 
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
  <modelVersion>4.0.0</modelVersion>
  <groupId>blog.testforge</groupId>
  <artifactId>testframework</artifactId>
  <version>1.0-SNAPSHOT</version>
  <build>
    <plugins>
    </plugins>
  </build>
</project>
```

## Add Dependencies

Add the necessary dependencies for Selenide, Allure, and other tools in your pom.xml file.

### Example pom.xml dependencies:

```xml
<dependencies>
  <!-- Selenide -->
  <dependency>
    <groupId>com.codeborne</groupId>
    <artifactId>selenide</artifactId>
    <version>6.16.0</version>
  </dependency>

  <!-- RestAssured -->
  <dependency>
    <groupId>io.rest-assured</groupId>
    <artifactId>rest-assured</artifactId>
    <version>5.3.0</version>
  </dependency>

  <!-- TestNG -->
  <dependency>
    <groupId>org.testng</groupId>
    <artifactId>testng</artifactId>
    <version>7.7.0</version>
    <scope>test</scope>
  </dependency>

  <!-- Allure TestNG -->
  <dependency>
    <groupId>io.qameta.allure</groupId>
    <artifactId>allure-testng</artifactId>
    <version>2.21.0</version>
  </dependency>
</dependencies>
```

## Add Steps and Tests

Create a simple test class to ensure your framework works correctly.

### Example test class:

```java
import com.codeborne.selenide.Selenide;
import org.testng.annotations.Test;
import static com.codeborne.selenide.Selenide.*;
import static com.codeborne.selenide.Condition.text;

public class GoogleSearchTest {
    @Test
    public void searchInGoogle() {
        open("https://google.com");
        $("input[name='q']").setValue("Selenide").pressEnter();
        $("#search").shouldHave(text("selenide.org"));
    }
}
```

## Add Reporter

Allure provides a visual way to review test results. Add the Allure TestNG listener to your tests.

### Add Allure configuration:

1. Add the listener in your testng.xml file:

```xml
<suite name="Test Suite">
  <listeners>
    <listener class-name="io.qameta.allure.testng.AllureTestNg" />
  </listeners>
  <test name="Google Tests">
    <classes>
      <class name="com.example.GoogleSearchTest" />
    </classes>
  </test>
</suite>
```

2. Add Maven plugin for Allure:

```xml
<plugin>
  <groupId>io.qameta.allure</groupId>
  <artifactId>allure-maven</artifactId>
  <version>2.11.2</version>
</plugin>
```

Generate the Allure report:

```bash
mvn allure:serve
```

## Package Fat JAR

A “fat JAR” bundles all dependencies into a single JAR file. This allows for easy execution.

### Configure Maven’s shade plugin:

```xml
<build>
  <plugins>
    <plugin>
      <groupId>org.apache.maven.plugins</groupId>
      <artifactId>maven-shade-plugin</artifactId>
      <version>3.4.1</version>
      <executions>
        <execution>
          <phase>package</phase>
          <goals>
            <goal>shade</goal>
          </goals>
        </execution>
      </executions>
    </plugin>
  </plugins>
</build>
```

### Package the JAR:

```bash
mvn clean package
```

## Execute from Docker

Run your tests in a Docker container for consistent environments.

### Dockerfile:

```dockerfile
FROM openjdk:17
WORKDIR /app
COPY target/testframework-1.0-SNAPSHOT.jar app.jar
ENTRYPOINT ["java", "-jar", "app.jar"]
```

### Build and run the container:

```bash
docker build -t testframework .
docker run testframework
```

## Bonus: Use Remote Browser

You can use a remote browser (e.g., Selenium Grid) for executing tests.

### Configure Selenide for a remote browser:

```java
import static com.codeborne.selenide.Configuration.*;

public class RemoteSetup {
    public static void setup() {
        remote = "http://<selenium-grid-ip>:4444/wd/hub";
        browser = "chrome";
        startMaximized = true;
    }
}
```

### Start Selenium Grid:

```bash
docker run -d -p 4444:4444 --name selenium-hub selenium/hub
docker run -d --link selenium-hub:hub selenium/node-chrome
```
